cubie.odesystems.symbolic.solver_helpers

Code generation for the linear operator β·M·v − γ·h·J·v.

The mass matrix M is provided at code-generation time either as a NumPy array or a SymPy matrix. Its entries are embedded directly into the generated device routine to avoid extra passes or buffers.

cubie.odesystems.symbolic.solver_helpers.generate_residual_code(equations: Iterable[Tuple[Symbol, Expr]], index_map: IndexedBases, M=None, is_stage: bool = True, func_name: str = 'residual_factory', cse: bool = True) → str[source]

Emit code for the residual factory.

Generates CUDA device code for computing residuals in Newton-Krylov ODE integration. The residual is computed as: beta * M @ v - gamma * h * (J @ eval_point)

Parameters:

  • equations (Iterable[Tuple[sp.Symbol, sp.Expr]]) – The differential equations defining the system.

  • index_map (IndexedBases) – Mapping of symbolic arrays to CUDA references.

  • M (array-like, optional) – Mass matrix. If None, uses identity matrix.

  • is_stage (bool, optional) – If True, generates stage residual (eval_point = base + a_ij * u). If False, generates end-state residual (eval_point = u).

  • func_name (str, optional) – Name of the generated factory function.

  • cse (bool, optional) – Apply common subexpression elimination.

Returns:

Generated CUDA device code for the residual factory.

Return type:

str
