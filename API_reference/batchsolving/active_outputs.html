ActiveOutputs

class cubie.batchsolving.arrays.BatchOutputArrays.ActiveOutputs(state: bool = False, observables: bool = False, state_summaries: bool = False, observable_summaries: bool = False, status_codes: bool = False)[source]

Bases: object

Track which output arrays are actively being used.

This class provides boolean flags indicating which output types are currently active based on array sizes and allocation status.

Parameters:

  • state (bool) – Whether state output is active.

  • observables (bool) – Whether observables output is active.

  • state_summaries (bool) – Whether state summaries output is active.

  • observable_summaries (bool) – Whether observable summaries output is active.

  • status_codes (bool) – Whether status code output is active.

observables: bool

update_from_outputarrays(output_arrays: OutputArrays) → None[source]

Update active outputs based on OutputArrays instance.

Parameters:

output_arrays – The OutputArrays instance to check for active outputs.

Returns:

Flags are updated in place.

Return type:

None

Notes

An output is considered active if the corresponding array exists and has more than one element (size > 1).
