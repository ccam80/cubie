src/cubie/outputhandling

Overview

The output handling package coordinates CUDA device callbacks that persist state trajectories and summary reductions from integration loops. It translates loop settings into validated configuration, compiles device functions through the CUDA factory infrastructure, and exposes sizing helpers so callers can allocate host and device buffers without duplicating dimension logic.

Entry point

OutputFunctions is the main interface. Instantiate it with loop dimensions and requested outputs to compile CUDA functions that save solver state, refresh summary metrics, and write reductions back to host arrays. The factory retains an OutputConfig instance and regenerates compiled callbacks when configuration changes.

Subpackages

summarymetrics houses the metric registry and built-in reductions. Import the subpackage to register CUDA metric implementations and access the shared registry via summary_metrics.

Public API

Configuration

OutputConfig

Validated configuration for solver outputs and summaries.

OutputCompileFlags

Boolean compile-time controls for CUDA output features.

Sizing utilities

SummariesBufferSizes

Buffer heights for summary metric staging buffers.

LoopBufferSizes

Staging buffer sizes consumed inside the integrator loop.

OutputArrayHeights

Heights of time-series and summary outputs.

SingleRunOutputSizes

Output array sizes for a single integration run.

BatchInputSizes

Input array sizes for batch integration runs.

BatchOutputSizes

Output array sizes for batch integration runs.

Runtime factories and registries

OutputFunctions

Factory that compiles and caches output management functions.

OutputFunctionCache

Cache container for compiled output functions.

summary_metrics

Registry and dispatcher for summary metrics.

register_metric

Create a decorator that registers a metric on instantiation.

Dependencies

  • Compiles CUDA callables through cubie.CUDAFactory.CUDAFactory and numba.cuda.

  • Loop buffers and output slices align with expectations from cubie.integrators.loops and related algorithm factories.
