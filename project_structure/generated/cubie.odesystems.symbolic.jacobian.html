cubie.odesystems.symbolic.jacobian._prune_unused_assignments(expressions: Iterable[Tuple[Symbol, Expr]], outputsym_str: str = 'jvp', output_symbols: Iterable[Symbol] | None = None) → List[Tuple[Symbol, Expr]][source]

Remove assignments that do not contribute to the final JVP outputs.

Parameters:

  • expressions – Topologically sorted assignments (lhs, rhs).

  • outputsym_str – Prefix identifying JVP output symbols. Ignored when output_symbols is provided.

  • output_symbols – Optional collection of output symbols to retain when pruning. When supplied, only assignments contributing to these symbols are kept.

Returns:

Pruned assignments that are required to compute the JVP outputs.

Return type:

List[Tuple[sp.Symbol, sp.Expr]]

Notes

The function assumes that the list is topologically sorted and that output assignments have left-hand-side symbols whose names start with "jvp[". It preserves the relative order of kept assignments.

Return symbolic assignments for the Jacobian-vector product (JVP).

Parameters:

  • equations – System equations including intermediates and outputs.

  • input_order – Mapping from each input symbol to its position in the input vector.

  • output_order – Mapping from each output symbol to its position in the output vector.

  • observables – Symbols to treat as auxiliary variables when constructing the JVP.

  • cse – Apply common-subexpression elimination before producing assignments.

Returns:

Symbolic assignments that compute the JVP for each output entry.

Return type:

List[Tuple[sp.Symbol, sp.Expr]]

Notes

Cached results are reused when possible and shared with the Jacobian cache through the key generated by get_cache_key().

Return the symbolic Jacobian matrix for the given equations.

Parameters:

  • equations – Full set of intermediate and derivative equations.

  • input_order – Mapping from each input symbol to its position in the input vector.

  • output_order – Mapping from each output symbol to its position in the output vector.

  • use_cache – Whether to reuse cached Jacobian computations when available.

  • cache_cse – Use the common-subexpression elimination form when creating cache keys.

Returns:

Generate the cache key from equations, orders, and the CSE flag.

Parameters:

  • equations – Equations expressed as (symbol, expression) pairs or a mapping of symbols to expressions.

  • input_order – Mapping from each input symbol to its position in the input vector.

  • output_order – Mapping from each output symbol to its position in the output vector.

Notes
